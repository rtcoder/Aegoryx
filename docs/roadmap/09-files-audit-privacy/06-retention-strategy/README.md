# Task: Retention Strategy

## Cel

Opisać i przygotować politykę retention, anonimizacji, eksportu i trwałego usuwania danych.

## Zakres

- Dokument decyzji.
- Kategorie danych.
- Proces hard delete po retention.

## Poza Zakresem

- Pełna automatyzacja GDPR v1.

## Zależności

- Files.
- Audit/activity.
- CRM sensitive fields.

## Kroki

- Sklasyfikować dane.
- Opisać różnicę soft delete vs permanent delete.
- Zaplanować joby cleanup.

## Subtaski

Brak.

## Acceptance Criteria

- Hard delete nie jest przypadkowy.
- Dane prywatne mają kierunek anonimizacji/usuwania.
- Audit zachowuje zgodność bez sekretów.
